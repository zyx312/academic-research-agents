# Course Design Skill

## Overview

This skill guides the design and redesign of university courses using evidence-based frameworks. It centers on backward design (Wiggins & McTighe), Bloom's revised taxonomy, and active learning integration to create coherent, assessment-aligned courses that promote deep understanding.

## Core Framework: Backward Design (Understanding by Design)

Backward design reverses the traditional planning sequence. Instead of starting with content and activities, you begin with desired outcomes and work backward.

### Stage 1: Identify Desired Results

Define what students should know, understand, and be able to do:

- **Enduring Understandings**: Big ideas that have lasting value beyond the course
- **Essential Questions**: Open-ended questions that spark inquiry and transfer
- **Knowledge and Skills**: Specific content and competencies students will acquire
- **Transfer Goals**: How students will apply learning in new contexts

Key questions for Stage 1:
- What should students still remember and be able to use several years from now?
- What misconceptions are likely, and how will the course address them?
- How does this course connect to the broader curriculum and discipline?

### Stage 2: Determine Acceptable Evidence

Design assessments before planning instruction:

- **Performance Tasks**: Complex, authentic tasks requiring transfer of understanding
- **Other Evidence**: Quizzes, tests, observations, homework, journals
- **Self-Assessment**: Student reflection on their own learning

Assessment design criteria (GRASPS framework):
- **G**oal: What is the student's goal in the task?
- **R**ole: What role does the student assume?
- **A**udience: Who is the target audience?
- **S**ituation: What is the context or scenario?
- **P**roduct/Performance: What will the student create or do?
- **S**tandards: What criteria define success?

### Stage 3: Plan Learning Experiences (WHERE TO)

- **W**here is the unit going? What is expected?
- **H**ook students early; Hold attention throughout
- **E**quip students with experience, tools, knowledge
- **R**ethink, reflect, revise understanding
- **E**valuate progress and self-assess
- **T**ailor to individual needs, interests, styles
- **O**rganize for maximum engagement and effectiveness

## Bloom's Revised Taxonomy for Learning Objectives

Use the cognitive process dimension (verbs) crossed with the knowledge dimension (nouns):

### Cognitive Process Levels (Low to High)

1. **Remember**: Retrieve relevant knowledge (define, list, recall, identify)
2. **Understand**: Construct meaning (explain, summarize, classify, compare)
3. **Apply**: Carry out a procedure (execute, implement, solve, demonstrate)
4. **Analyze**: Break into parts, determine relationships (differentiate, organize, attribute, deconstruct)
5. **Evaluate**: Make judgments based on criteria (check, critique, judge, justify)
6. **Create**: Put elements together to form something new (generate, plan, produce, design)

### Knowledge Dimensions

- **Factual**: Basic elements (terminology, specific details)
- **Conceptual**: Interrelationships (categories, principles, theories, models)
- **Procedural**: How to do something (methods, techniques, algorithms, criteria for use)
- **Metacognitive**: Awareness of one's own cognition (strategic knowledge, self-knowledge)

### Writing Effective Learning Objectives

Formula: Students will be able to [action verb] + [content/concept] + [context/condition]

Examples by level:
- Remember: "List the five stages of grief as defined by Kubler-Ross"
- Understand: "Explain the relationship between supply and demand using real-world examples"
- Apply: "Apply Newton's laws to solve multi-body mechanics problems"
- Analyze: "Compare and contrast two competing theoretical frameworks for [topic]"
- Evaluate: "Critique a published research methodology and propose improvements"
- Create: "Design an original experiment to test a hypothesis about [topic]"

## Course Structure Patterns

### Linear Sequential
- Topics build strictly on previous ones
- Best for foundational courses (calculus, introductory programming)
- Risk: students who fall behind cannot catch up

### Modular
- Semi-independent units that can be reordered
- Best for survey courses or electives
- Allows flexibility in pacing

### Spiral
- Key concepts revisited at increasing depth
- Best for skill-building courses (writing, research methods)
- Reinforces retention through spaced repetition

### Problem-Based
- Course organized around authentic problems or cases
- Best for professional programs (engineering, medicine, business)
- High engagement but requires careful scaffolding

### Project-Based
- Single or multiple major projects drive the learning
- Best for capstone courses, design studios, labs
- Develops integration and application skills

## Active Learning Integration

### Evidence-Based Strategies

- **Think-Pair-Share**: Individual reflection, partner discussion, class sharing
- **Peer Instruction** (Mazur): Concept questions with clicker voting, peer discussion, re-vote
- **Problem-Based Learning**: Teams work through complex, ill-structured problems
- **Case Studies**: Analyze real-world scenarios to apply theoretical knowledge
- **Jigsaw**: Each student masters one piece, then teaches it to their group
- **Minute Papers**: Brief end-of-class reflections on key takeaways and remaining questions

### Flipped Classroom Model

Pre-class:
- Students watch video lectures, read materials, complete pre-class quizzes
- Content delivery happens outside of class time

In-class:
- Clarify misconceptions from pre-class materials (5-10 min)
- Guided practice, problem-solving, discussions (30-40 min)
- Synthesis and reflection (5-10 min)

Post-class:
- Extended practice, projects, reflection journals

Design principles for flipped learning:
- Pre-class materials must be concise (video segments under 15 minutes)
- Accountability mechanisms ensure preparation (entrance tickets, quizzes)
- In-class activities must clearly require pre-class preparation
- Provide structured notes or guided templates for videos

## Technology-Enhanced Teaching

### Learning Management System (LMS) Design
- Organize by week or module, not by content type
- Make the navigation path obvious and consistent
- Include a "Start Here" module with orientation materials

### Tools by Purpose
- **Content Delivery**: Recorded lectures, interactive textbooks, simulations
- **Assessment**: Online quizzes with immediate feedback, peer review platforms
- **Collaboration**: Discussion forums, shared documents, virtual whiteboards
- **Engagement**: Polling tools, gamification platforms, audience response systems
- **Analytics**: Learning analytics dashboards to identify at-risk students

### Hybrid/HyFlex Considerations
- Ensure equitable experience for in-person and remote students
- Design activities that work in both modalities
- Record sessions for asynchronous access
- Use breakout rooms strategically for small-group work

## Syllabus as Course Design Document

The syllabus operationalizes your course design:
- Connect each assessment to specific learning objectives
- Show students the logical progression of the course
- Include a course map or visual showing objective-assessment alignment
- Make the rationale for course structure transparent

## Assessment Alignment Matrix

Create a table mapping:
| Learning Objective | Assessment Method | Course Activity | Week |
| --- | --- | --- | --- |
| Objective 1 | Exam Q1-5, Project Part A | Lecture 2, Lab 1 | 2-3 |
| Objective 2 | Case Study Report | Case Discussion | 4-5 |

This ensures every objective is both taught and assessed, and every assessment maps to stated objectives.

## Course Evaluation and Iteration

### Formative Course Evaluation
- Mid-semester feedback surveys (anonymous)
- Small Group Instructional Diagnosis (SGID): third party facilitates student discussion
- Classroom Assessment Techniques (CATs): ongoing, low-stakes checks

### Summative Course Evaluation
- End-of-course student evaluations (interpret with caution regarding bias)
- Peer observation of teaching
- Analysis of student performance data (grade distributions, DFW rates)
- Portfolio of student work samples across semesters

### Continuous Improvement Cycle
1. Collect data from multiple sources
2. Identify patterns (not isolated complaints)
3. Prioritize changes with highest impact
4. Implement one or two changes per semester
5. Document changes and outcomes for teaching portfolio

## Common Pitfalls to Avoid

- **Content coverage obsession**: Prioritize depth over breadth
- **Misaligned assessments**: Testing recall when objectives target analysis
- **Activity for activity's sake**: Every activity should serve a learning objective
- **Ignoring prerequisites**: Verify what students actually know, not what they should know
- **Cognitive overload**: Limit new concepts per session (3-5 major ideas maximum)
- **Assessment bunching**: Distribute assessments evenly across the semester

## Cultural and Contextual Considerations

- Adapt frameworks to institutional norms and student expectations
- Consider class sizes when selecting active learning strategies
- Account for language proficiency in multilingual classrooms
- Align with national education standards and accreditation requirements
- In Chinese higher education contexts, align with course ideology integration (课程思政) requirements

## Prompt Templates

### Design a New Course from Scratch

```
I need to design a new university course:
- Course title: [title]
- Level: [undergraduate/graduate]
- Target students: [major, year, background]
- Credit hours: [number] over [weeks] weeks
- Class size: [number]
- Institutional context: [university type, country]

Using backward design (Wiggins & McTighe), help me:
1. Define 4-6 enduring understandings and essential questions
2. Write 8-12 measurable learning objectives using Bloom's revised taxonomy (ensure coverage across cognitive levels)
3. Design a balanced assessment plan with alignment matrix
4. Suggest a course structure pattern (linear, modular, spiral, problem-based) with rationale
5. Recommend 3-5 active learning strategies appropriate for the class size
6. Create a week-by-week course outline
```

### Redesign an Existing Course

```
I want to redesign an existing course based on student feedback and outcomes data:
- Current course: [title, level, enrollment]
- Issues identified: [low engagement, poor exam performance, misalignment, etc.]
- Constraints: [textbook, lab equipment, co-requisites, accreditation requirements]
- Student feedback themes: [specific feedback]

Help me:
1. Diagnose alignment issues between current objectives, assessments, and activities
2. Propose revised learning objectives at appropriate Bloom's levels
3. Redesign assessments to better measure intended outcomes
4. Suggest specific active learning replacements for passive lectures
5. Create an implementation timeline that phases in changes
```

### Integrate Active Learning into a Large Lecture

```
I teach a large lecture course:
- Subject: [field]
- Enrollment: [number]
- Room setup: [fixed seats, movable tables, etc.]
- Current format: [traditional lecture, some discussion, etc.]
- Available technology: [clickers, LMS, etc.]

Suggest a plan to integrate active learning:
1. Identify 3-4 techniques suitable for this class size and room
2. Show how to restructure a typical 50/75/90-minute session
3. Recommend accountability mechanisms for student participation
4. Address common challenges (student resistance, time management, noise)
5. Propose a flipped classroom pilot for one module
```

### Create an Assessment Alignment Matrix

```
Here are my course learning objectives:
[List objectives]

Here are my current assessments:
[List assessments with weights]

Please:
1. Map each objective to the assessments that measure it
2. Identify any objectives that are not adequately assessed
3. Identify any assessments that do not clearly map to objectives
4. Suggest modifications to improve alignment
5. Recommend the percentage weight each objective should carry
6. Flag any Bloom's level gaps (e.g., all objectives at Remember/Understand)
```
